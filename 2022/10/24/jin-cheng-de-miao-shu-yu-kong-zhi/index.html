进程控制

基本概念：进程控制就是要实现进程状态的转换。进程控制用原语实现（原语用开/关中断来实现，是一种特殊的程序，原语执行必须一气呵成，不能中断）

graph LR

a---e[进程的唤醒]
a---f[进程的切换]
e & d ---g[阻塞和唤醒要成对出现]

进程通信

概念：指进程之间的信息交换

进程通信方式：共享储存、管道通信、消息传递

线程、多线程模型

线程的重要属性：线程是处理机调度的单位，进程是资源分配的单位。同一进程的各线程共享进程拥有的资源，同一进程内的线程切换不会导致进程切换。

线程的实现方式：用户级线程、内核级线程、组合方式

多线程模型：用户级线程向内核级线程映射方式

  1. 多对一模型
  2. 一对一模型
  3. 多对多模型

处理机调度

调度的三个层次（硬盘–内存–CPU)

高级调度（外存->内存）

高级调度主要指调入问题，因为调入的时机由操作系统来确定，但调出的时机必然是作业运行结束才调出。

中级调度（外存->内存）

引入虚拟储存之后，可将暂时不能运行的进程调至外存等待，等到宠重新具备了运行条件且内从稍有空闲，再重新调入内存。这样提高了内存利用率和系统吞吐量。调到外存的进程状态为挂起状态。

低级调度（进程调度，内存->CPU）

从就绪队列中选取一个进程为其分配处理机。

进程调度的方式

抢占式和非抢占式

调度算法的评价指标

周转时间：作业完成时间-作业提交时间

平均周转时间：$\frac{各作业周转时间之和}{作业数}$

带权周转时间：$\frac{作业周转时间}{作业实际实际使用时间}$

调度算法

先来先服务（FCFS）

先到先调度，非抢占式算法

优点：公平、算法简单

缺点：排在长作业后的短作业需要等待很长时间，对短作业不友好，带权周转时间长

短作业优先（SJF）

非抢占式算法：选择先到达的运行时间最短的先调度（默认）

抢占式算法：选择先到达的运行时间最短的先调度，若比当时运行的剩余时间更短，则抢占优先调度。

优点：平均等待、周转时间最短

缺点：对长作业不友好，会导致长作业饥饿

高响应比优先（HRRN）

非抢占式

响应比=$\frac{等待时间+要求服务时间}{要求服务时间}$>=1

综合考虑等待时间和运行时间，避免长作业饥饿

时间片轮转调度算法（RR）

抢占式

轮流让就绪队列中的进程依次执行一个时间片，时间片不能太大或太小

优点：公平、响应快

不会导致饥饿

优先级调度算法

抢占式、非抢占式

当优优先级相同时，先到的先上处理机。

会导致饥饿。

多级反馈队列调度算法

对上述各种算法的折中权衡，用于进程调度

抢占式算法

设置多级就绪队列，各级队列优先级由高到低，时间片由小到大。新进程到达时先进入第一级队列，时间片结束时进入下一级队列，若在最后一级，则放入该级队列队尾。新进程优先级高，会抢占当前处理机。

优点：对各类进程相对公平（FCFS的优点）；每个新到达的进程都可以很快得到响应(RR优点)；短进程只用较少的时间就可以完成（SPF优点）；不必实际估计进程运行时间；可灵活地调整对各类进程的偏好程度。

当有短进程不断加入时，会导致下级进程饥饿。

进程互斥、同步

进程互斥

graph LR
a[两种资源共享方式]---b[互斥共享方式]
a---c[同时共享方式]
b-->d["一个时间段内只允许一个进程访问该资源(临界资源)"]

对临界资源的互斥访问逻辑

do{
    entry section; // 进入区，上锁
    critical section; //临界区 访问
    exit section; //退出区 解锁
    remainder section; //剩余区 其他
} while(true)

互斥访问原则：

  1. 空闲让进
  2. 忙则等待
  3. 有限等待
  4. 让权等待

进程互斥的软件实现方法

  1. 单标志发

    算法思想：两个进程在访问临界区后会把使用临界区的权限转交给拎一个进程。也就是说每个进程进入临界区的权限只能被另一个进程赋予。

    int turn=0; //turn 表示当前允许进入临界区的进程号
//p0:
while(turn!=0);   //进入区
critical section;	//临界区
turn=1;		//退出区
remainder section;	//剩余区

//p1:
while(turn!=1);
critical section;
turn=0;
remainder section;

    特点：对临界区的访问，一定是按顺序交替进行，若p0一直不访问临界资源，则p1也一直不能访问。

    存在的问题： 违背”空闲让进“原则

  2. 双标志先检查

    算法思想：设置bool型数组flag[ ]，标志各个进程进入临界区的意愿。

    bool falg[2];
falg[0]=false;
flag[1]=false;
// 先检查，后上锁
//p0:
while(flag[1]);  1
flag[0]=true;	2
critical section;
flag[0]=false;
remainder section;


//p1:
while(flag[0]);	3
flag[1]=true;	4
critical section;
flag[1]=false;
remainder section;

    存在的问题：进程执行顺序不可控，若语句执行顺序为：1->3->2->4，则违反”忙则等待“原则。出现这种问题的原因时进入区的”检查“和”上锁“不能一气呵成的执行。

  3. 双标志后检查

    算法思想：双标志先检查的改版

    bool falg[2];
falg[0]=false;
flag[1]=false;
// 先上锁，后检查
//p0:
flag[0]=true;  1
while(flag[1]);	2
critical section;
flag[0]=false;
remainder section;

//p1:
flag[1]=true;	3
while(flag[0]);	4
critical section;
flag[1]=false;
remainder section;

    存在的问题：若按照1->3->2->4的顺序，p0和p1都将无法进入临界区，虽解决了”忙则等待“的问题，但是违背了”空闲让进“原则。

  4. Peterson算法

    算法思想：当进程都争取临界区的时候，使其主动让对方先使用临界区

    bool flag[2];	//		临界区意愿，初始值为false
int turn=0;	//优先使用临界区的进程

//p0:
flag[0]=true;
turn=1;
while(flag[1]&&turn==1);
critical section;
flag[0]=false;
reminder section;
//p1:
flag[1]=true;
turn=0;
while(flag[0]&&turn==0);
critical section;
flag[1]=false;
reminder section;

    Peterson算法解决了进程互斥问题，遵循了空闲让进、忙则等待、有限等待三个原则，但是当当前进程不使用临界资源时，仍然会占用CPU进行无意义的while检查，没有遵循让权等待的原则。

进程互斥的硬件实现方法

  1. 中断屏蔽方法

    原理：利用”开/关中断指令“实现，与原语实现思想相同，当某进程开始访问临界区到访问结束都不允许被中断，也就不能发生进程切换。

    优点：简单、高效

    缺点：不适用多处理解，开/关中断指令只能允许在内核态，权限大，只适用于操作系统内核进程。

  2. TestAndSet指令

    简称TS指令、TSL指令，由硬件实现，执行过程不允许被中断。

    //lock表示当前临界区是否被加锁，true表示已加锁
bool TestAndSet(bool *lock){
    bool old;
    old=*lock;
    *lock=true;
    return old;
}
//实现互斥算法
while(TestAndSet(&lock));
//临界区代码段
lock=false;

  3. Swap指令

    Swap指令，由硬件实现，执行过程不允许被中断。逻辑如下：

    //Swap 指令的作用是交换两个变量的值
Swap(bool *a,bool *b){
    bool temp;
    temp=*a;
    *a=*b;
    *b=temp;
}
//算法逻辑 lock 表示当前临界区是否被加锁
bool old=true;
while(old==true)
    Swap(&lock,&old);
//临界区代码段
lock=false;

信号量机制

信号量是一种整数或记录型的变量，用来表示系统中某资源的数量。用户可以使用操作系统提供的一原语wait(S) P(S)、signal(S) V(S)来对信号量进行操作，从而实现进程互斥、同步问题。

整型信号量

int S=1;  //表示一个资源
viod wait(int S){  
    while(S<=0);  
    ！！当不允许进入临界区的时候循环等待，又不能被中断，不是应该卡死在这里吗？？
    S=S-1;
}
void signal(int S){
    S+=1;
}

//pn:
...
wait(S);	//进入临界区
//使用临界资源
signal(S);	//退出临界区
...

会忙等，不满足让权等待原则。

记录型信号量

typedef struct{
    int value;	//剩余资源数
    struct process *L;	//等待队列
} semaphore;

void wait(semaphore S){
    S.value--;
    if(S.value<0){
        block(S.L);	//block原语，使该进程从运行态进入阻塞态，并挂到等待队列
    }
}

void signal(semaphore S){
    S.value++;
    if(S.value<=0){
        wakeup(S.L);	//释放资源后，还有别的进程等待资源，则使用wakeup原语唤醒一个进程使用刚才的资源，该进程从阻塞态变为就绪态
    }
}

解决忙等问题，遵循让权等待的原则。

生产者消费者问题

semaphore mutex=1;
semaphore full=0;
semaphore empty=n;
void producer(){
    while(true){
        //生产一个产品
        wait(empty);
        wait(mutex);
        //放入缓冲区
        signal(mutex);
        signal(full);
    }
}

void consumer(){
    while(true){
        wait(full);
        wait(mutex);
        //从缓冲区取走一个产品
        signal(mutex);
        signal(empty);
        //使用产品
    }
}

当每个哲学家都先拿左边再拿右边筷子时，会出现每个人只拿到左边筷子的情况，这时将会出现死锁。

解决方案：

  1. 每次只允许四位哲学家同时拿筷子。

    semaphore chopstick[5]={1,1,1,1,1};
semaphore mutex=4;
void pi(){
    while(true){
        wait(mutex);
        wait(chopstick[i]);
        wait(chopstick[(i+1)%5]);
        signal(mutex);
        //吃饭
        signal(chopstick[i]);
        signal(chopstick[(i+1)%5]);
        //思考
    }
}

  2. 奇数号哲学家先拿左边筷子，再拿右边筷子，偶数号反之。

    semaphore chopstick[5]={1,1,1,1,1};
void pi(){
    while(true){
        if(i%2){
            wait(chopstick[i]);
            wait(chopstick[(i+1)%5]);
        }
        else{
            wait(chopstick[(i+1)%5]);
            wait(chopstick[i]);
        }
                //吃饭
        signal(chopstick[i]);
        signal(chopstick[(i+1)%5]);
        //思考
    }
}
